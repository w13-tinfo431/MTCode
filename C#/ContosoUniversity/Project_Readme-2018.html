Modify the Contoso University website application in the following ways:

  1. 10 pts: Change the site's theme using Bootstrap. Your choice of themes, but it must be changed to one of the selections. You must further modify your theme in some arbitrary way that makes it unique in the class. If you arbitrarily modify 4 or 5 items in your theme, it becomes highly unlikely that your theme will line up exactly with another student website theme.
  2. 10 pts: Use NuGet Package Manager to add ELMAH (get the application-wide package) error logging to your application. You are not implementing logging. This assignment item is to get and install the package.
  3. 10 pts: Add a real-time web component with ASP.NET SignalR. This tutorial is a complete and separate client-server chat appication. The tutorial does not complete the step of linking the chat application back into your Contoso University application. This item requires that you integrate the live chat application into your Contoso University website in a way that is intuitive and user-friendly. An example of friendly integration would be having a link to launch the chat feature. I would not expect to have to type in a URL to reach this feature.
  4. Object-Oriented Stuff - Using a code first approach, add the following properties and Model(s) to represent the described changes. For full credit on these, added data items should be propagated to corresponding Views. You can consider the user of this application to be authorized to view all data in the database.
    1. 10 pts: A Person "has a" Address object (use composition for this). An Address object has a single property of type string for Email (for simplicity that's all an Address has)
    2. 10 pts: A Student "has a" CreditsEarned property
    3. 10 pts: A Student "has a" Gpa property that can express overall grades with exactly 2 decimal points, such as 3.87, etc.
    4. 10 pts: Use data attributes to insure that Email addresses entered into the system conform with the pattern [username]@contoso.edu. Hint: Refer to the regex material in the Galloway book.
    5. 10 pts: Use data attributes to insure that Gpa is expressed as a value in range from 0.7 and 4.0 inclusive.
  5. 10 pts: The Students (Index) view should be sortable ascending and descending according to the LastName column or the GPA column.
  6. 10 pts: The final 10 points is a catch all category that includes final prep of the application for testing. Remember to update the initialization of test data records in the Seed() method so that you are providing a complete and ready to go website for testing all the added features. Remember to include a nice friendly (and obvious) user interface for the chat client app.
